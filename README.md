# 本地 AI 聊天

一个面向本地模型的单页聊天工作台。

它提供聊天、会话管理、人设模板、设定文件夹、技能启用、定时任务、文件工具与代码展示等能力，适合在本地模型服务基础上做持续对话和轻量自动化。

## 启动方式

在当前目录执行：

```powershell
node server.js
```

启动后访问：

`http://127.0.0.1:8000`

默认会通过同源代理把请求转发到本地模型服务：

- 聊天接口：`/api/v1/chat/completions`
- 模型列表：`/api/v1/models`
- 默认目标：`http://127.0.0.1:1234`

## 主要能力

### 1. 聊天与会话

- 社交软件风格聊天界面，支持 AI / 用户头像上传
- 用户消息、AI 回复均显示时间，精确到秒
- 新建会话会重置上下文，不带入上一条会话历史
- 当前会话自动保存，标题默认取第一条用户消息的第一句话
- 支持切换历史会话，并恢复该会话聊天内容用于后续上下文
- AI 正在输入时，禁止切换会话或执行会话相关操作
- 支持删除某一轮“用户问题 + AI 回答”，并在限定时间内撤销

### 2. 连接与模型

- 测试本地模型连接
- 读取模型列表并切换当前模型
- 记录模型选择历史
- 顶部显示上下文字符、预估 Prompt、Total、速率、上下文使用情况等运行指标
- 上下文使用情况以 `Total` 为主进行计算
- 超过设置的总上下文上限 Token 时，会显示警示状态

### 3. 人设与设定

- 支持直接编辑系统提示词和人设内容
- 支持从文件导入 / 导出 / 清空人设
- 模板选择为“选择即应用”，不需要额外点击应用按钮
- 支持读取工作区 `人设` 目录下的 `.md` 文件作为工作区人设模板
- 会记住上次选择的人设模板，刷新后自动恢复
- 支持导入“设定文件夹”，适合小说设定、世界观、角色设定等长期上下文
- 设定文件夹内容会持久化保存，并在后续对话中自动带入

### 4. Skills

- 支持读取工作区与 Codex 技能目录中的技能
- 技能读取只读取 `SKILL.md`
- 技能启用后，仅按 `SKILL.md` 中的要求参与后续对话
- 支持取消启用已启用技能
- 技能读取状态和启用状态会持久化保存，刷新后自动恢复

### 5. 文件与工具

- 支持粘贴、拖拽、上传图片和常见文本文件
- 图片附件横向排列，支持双击查看大图
- 聊天中的工具活动以小按钮形式显示，点击弹框查看详细记录
- 默认不会自动写入本地文件
- 只有你明确要求“保存到本地 / 写入文件”等，才允许写文件类工具执行
- 临时文件统一约束在当前目录下的 `temp-files/`

### 6. 代码显示

- 支持大多数 Markdown fenced code block 展示
- 常见语言如 `python`、`js`、`ts`、`html`、`css`、`java`、`go`、`rust`、`sql`、`bash` 等都可正常显示
- 代码块支持复制
- 复制按钮固定在代码块头部，不跟随代码区域滚动消失
- 长代码默认完整展开显示，主要保留横向滚动

### 7. 定时任务

- 支持在界面中创建、编辑、删除、启停、立即执行定时任务
- 统一使用 Cron 表达式配置执行时间
- 任务执行模型默认使用当前基础配置中的模型选择
- 可通过自然语言让模型创建定时任务
- 定时任务结果默认以普通 AI 回复的方式推送到当前聊天会话
- 若当前没有可用会话，会自动创建会话承接任务结果
- 定时任务数据保存在 `scheduled-tasks.json`

## 使用说明

### 人设模板

- 内置模板和工作区模板会一起显示在“模板选择”中
- 选择某个人设模板后会立即覆盖到人设输入框
- 当前选择会保存到本地，刷新后自动恢复

### 设定文件夹

- 适合导入小说项目、角色设定、世界观、章节规则等文本资料
- 导入后会自动摘要并保存在本地配置中
- 后续对话会自动把这些设定作为长期上下文带入

### Skills

- 读取技能后会展示技能列表和 `SKILL.md` 预览
- 启用后，后续对话只按 `SKILL.md` 要求执行
- 取消启用后，不再继续带入该技能上下文

### 定时任务

- 推荐使用标准 5 段 Cron 表达式
- 例如每天早上 9 点：`0 9 * * *`
- 定时任务可在左侧卡片内直接编辑，不需要弹窗

## 当前约束

- 本项目默认依赖本地模型服务运行在 `http://127.0.0.1:1234`
- 某些新加的接口和功能在修改后需要重启 `node server.js` 才会生效
- 如果任务内容需要实时天气等外部信息，仍依赖服务端已接入的对应工具能力

## 建议

- 修改 [app.js](/e:/AI文件/AI%20web/app.js) 或 [server.js](/e:/AI文件/AI%20web/server.js) 后，优先重启本地服务再刷新页面
- 如果发现界面状态和预期不一致，先确认当前访问的是最新启动的 `node server.js` 进程
