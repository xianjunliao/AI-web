# novel_writer 提示词模板

这份模板只保留高频必填项，默认适配当前 skill 的规则：

- 正式角色姓名由用户决定
- 正式正文不要小标题
- 场景切换要自然
- 阅读体验贴近当前主流网文
- 写作类模板默认严格按 `SKILL.md` 说明落盘到本地，不只返回文本结果

如果项目目录里已经有 `world.md`、`characters.md`、`outline.md`、`memory/novel_memory.md`，就优先让 skill 直接读取，不要重复粘贴长设定。

## 通用原则

- 本文件中的写作类模板默认按“保存到本地”使用
- 想触发保存脚本，就明确写：`严格按 JSON 结构输出`
- 保存时，`content` 只放正文本体，不要把章节标题再写进去
- 角色名字如果还没定，就写：`名字由我指定，你只能推荐，不能直接使用`
- 想让执行更稳，开头先加这句：`请严格按 novel_writer skill 的说明读取、生成并落盘，不要只回答，要实际保存到本地。`

```text
角色命名方式：
- 名字由我指定，你只能推荐，不能直接使用

我的想法：
- 由我补充 3 到 6 条

要求：
- 设定要可写，不要空
- 人物要有缺陷和冲突
- 总纲要能支撑连载

输出要求：

```
