# novel_writer 长篇模板

这份模板适合 10 万字以上长篇。思路很简单：

1. 先定设定和总纲
2. 再定分卷和前期章节
3. 然后逐章推进
4. 每隔几章做一次连贯性检查


## 通用原则

- 本文件中的写作类模板默认按“保存到本地”使用
- 想触发保存脚本，就明确写：`严格按 JSON 结构输出`
- 保存时，`content` 只放正文本体，不要把章节标题再写进去
- 角色名字如果还没定，就写：`名字由我指定，你只能推荐，不能直接使用`
- 想让执行更稳，开头先加这句：`请严格按 novel_writer skill 的说明读取、生成并落盘，不要只回答，要实际保存到本地。`

## 阶段 1：先搭长篇骨架

```text
请使用 novel_writer skill 规划一部长篇小说。

任务类型：世界观设定 + 人物设定 + 故事总纲 + 分卷规划 + 章节大纲
项目名：my_long_novel
题材：由我补充

目标规模：
- 目标总字数：120000
- 卷数：4
- 每卷章节数：12 到 18
- 单章字数：2000 到 3000

角色命名方式：
- 名字由我指定，你只能推荐，不能直接使用

我的核心想法：
- 由我补充 5 到 10 条

要求：
- 主线要强
- 人物要有成长弧和冲突
- 中段不能塌
- 伏笔尽量前置规划
- 设定要能支撑连载

输出内容：
- 世界观设定
- 主角、关键配角、反派设定
- 故事主题
- 三幕式总纲
- 分卷规划
- 前 10 到 20 章大纲
```

## 阶段 1 补充：完整版长篇规划模板

这份模板适合你已经有比较完整的题材想法，想一次性把世界观、人物、总纲、分卷和前期章节全搭起来。

```text
请使用 novel_writer skill 帮我规划并创作一部长篇小说。

任务类型：世界观设定 + 人物设定 + 故事总纲 + 分卷规划 + 章节大纲
项目名：my_long_novel
题材：由我补充

目标规模：
- 目标总字数：120000
- 卷数：4
- 每卷章节数：15 章左右
- 单章字数：2000 到 3000
- 每 3 章左右安排一个小高潮
- 每卷结尾要有重大转折或情感爆点

角色命名方式：
- 主角姓名：由我补充
- 关键配角姓名：由我补充
- 反派姓名：由我补充
- 名字由我指定；如果我没定，你只能推荐，不能直接使用

我的核心故事设定：
- 由我补充 6 到 12 条

我希望的主题：
- 由我补充 3 到 6 条

世界观方向：
- 时间背景：由我补充
- 社会状态：由我补充
- 技术背景：由我补充
- 核心矛盾：由我补充
- 法律与秩序：由我补充
- 更深层设定：由我补充

主角设定要求：
- 主角必须是：由我补充
- 性格复杂度要求：由我补充
- 初始状态：由我补充
- 角色成长弧：由我补充
- 悲剧走向或最终抉择：由我补充

核心角色设定要求：
- 关键角色 1：由我补充
- 关键角色 2：由我补充
- 关键角色与主角的关系变化：由我补充
- 自我意识、堕落弧或执念演化逻辑：由我补充

反派与对立面要求：
- 不要脸谱化
- 要有自己的逻辑和立场
- 可以包含企业、政府、安全部门、伦理组织等多方博弈
- 冲突要体现价值观碰撞，而不只是追杀关系

剧情结构要求：
- 第一卷：由我补充
- 第二卷：由我补充
- 第三卷：由我补充
- 第四卷：由我补充

情绪和风格要求：
- 整体气质：由我补充
- 前期氛围：由我补充
- 中期氛围：由我补充
- 后期氛围：由我补充
- 结尾气质：由我补充
- 正式正文不要小标题
- 场景切换自然
- 阅读体验贴近当前主流网文
- 不要写成：由我补充 2 到 4 条

输出内容：
- 世界观设定
- 主角设定
- 核心角色设定
- 关键配角与对立阵营设定
- 故事一句话概述
- 故事主题
- 三幕式总纲
- 4 卷分卷规划
- 前 20 章详细大纲

要求：
- 主线必须清晰
- 人物必须有成长弧和堕落弧
- 关键意识或情感演化过程必须可信
- 主角与核心角色的感情推进必须层层递进，不能一开始就太满
- 后期毁灭性结局必须有充分铺垫
- 最终超脱式结局必须与前面巨大悲剧形成强烈对照
- 整部小说必须可写、可连载、可持续推进

输出要求：
- 直接输出结果
- 如果需要落盘保存，再严格按 novel_writer skill 的 JSON 结构输出
```

## 阶段 2：整理成长篇记忆

```text
请继续使用 novel_writer skill 整理这部长篇的项目记忆。

任务类型：整理小说记忆
项目名：my_long_novel

已有内容：
- 直接读取当前项目中的设定与总纲

- 提炼长期续写必须保留的信息
- 标出硬设定、主线目标、角色关系和已规划伏笔

- 直接输出结果，不保存
```

## 阶段 5：每隔几章做一次检查

```text
请使用 novel_writer skill 检查最近几章的连贯性问题。

任务类型：连贯性检查
项目名：my_long_novel

已有设定：
- 直接读取当前项目里的设定和写作记忆

待检查内容：
- 最近 3 到 5 章摘要或正文

检查重点：
- 人物行为
- 时间线
- 伏笔回收
- 节奏是否失衡
- 是否偏离主线

输出要求：
- 优先列问题
- 每个问题包含：问题、原因、修复建议、影响范围
```

## 使用建议

- 先定骨架，再写正文，不要一上来就整本硬写
- 如果项目里已有设定文件，后续提示词尽量写“直接读取当前项目”
- 如果名字还没定，就只让它推荐，不要让它直接命名
